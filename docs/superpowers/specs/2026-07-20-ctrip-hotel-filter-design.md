# 携程酒店软筛选与现爬合并设计

**日期：** 2026-07-20  
**状态：** 实现中  

## 目标

完整生成时每次现爬携程酒店（不缓存），按软规则排序后优先并入住宿候选池，不足用高德补齐。失败则降级高德并标记「酒店数据未更新」。

## 规则（软排序，不硬淘汰）

- 好评率：≥90% 高分；未知降权  
- 开业：近 5 年高分；未知降权  
- 华住会品牌（全季/汉庭/桔子/宜必思/漫心/海友等）加分  
- 近地铁、交通便利文案加分  

## 流程

1. 高德 POI 池照旧  
2. `search_ctrip_hotels(destination)` 列表 + 抽样详情 → 打分排序  
3. 高德关键词补坐标 → 转 `Poi` 插到 `pool['hotel']` 最前并去重  
4. 落库 `hotel_fetch_status`=`ok`|`amap_only`，`hotel_candidates` 精简列表  
5. Prompt 说明住宿列表已按上述偏好软排序  

## 非目标

登录态、硬过滤清空结果、按天重生重爬酒店。
